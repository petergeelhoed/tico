# Contributing to Tico

Thank you for considering contributing to the Tico project! We appreciate your help in making this project better for everyone.

## Commit Message Standards

We follow the [Conventional Commits](https://conventionalcommits.org/en/v1.0.0/) specification to make it easier to write and understand our commit messages.

### Standard Format

A commit message should be structured as follows:

```
mood(scope): subject

[optional body]

[optional footer]
```

- **mood**: A noun describing the type of change (e.g., feat, fix, docs, style, refactor, perf, test, chore).
- **scope**: A noun describing the area of the codebase or process (optional).
- **subject**: A short description of the change (max of 72 characters).
- **body**: A more detailed explanation of the change (if necessary).
- **footer**: Any information about breaking changes or issues addressed.

### Example Commit Messages

- `feat(auth): add JWT authentication`
- `fix(api): resolve 404 error on missing resource`
- `docs(README): update installation instructions`
